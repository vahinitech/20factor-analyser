Demo Playbook

Run the stall demo in
under 60 seconds per visitor.

Everything you need to show Vahini at an event, school visit or investor stall, the booth flow, the kit, the easy interface, and the best ways to share the report. No backend required for the basic demo.

1The booth flow

A visitor walks up. Here is the whole loop, paper in, report out.

1

Hand them a sheet

Give the printed A4 practice sheet, a reference sentence on top, ruled space to write below.

2

They write

They copy the sentence in their normal handwriting. Takes 30–40 seconds.

3

Snap a photo

On the booth phone/tablet, open Studio → fill name → Photo / Scan → take the picture.

4

Report appears

The 20-factor report renders on screen in seconds. Show it, then Save as PDF or share the link.

Want the “wow”? For VIPs, switch to the Battu mode and let them write with the IMU pen, the live 208 Hz force/tilt/velocity stream on screen is the showstopper, then “Finish” generates the pen report.

2The booth kit

Minimal hardware. Everything below runs offline from the booth device.

1 × tablet or laptop 1 × phone camera (or the tablet’s) Printed practice sheets A pen or two Optional: IMU pen for the hero demo Optional: PaddleOCR server for live recognition
  • Practice sheets, print Vahini Practice Sheet.html (the “Switch sentence” button cycles three options). Print 30–50 for a busy day.
  • The interface, open Vahini Analyser.html full-screen. That is the entire app; no install, no login.
  • Stand mount, a phone stand at fixed height over a contrasting mat gives clean, repeatable photos.

3The one-tap interface

Built so a volunteer can run it with zero training.

For the operator

  • Open Studio → type the visitor’s first name → Continue.
  • Pick Photo / Scan → tap the dropzone → take the photo.
  • The reference sentence is pre-selected, no typing the expected text.
  • Tap Run analysis. Report is ready in a few seconds.

Photo tips for clean scores

  • Fill the frame with the writing area only; avoid the table.
  • Even light, no harsh shadow, the engine auto-switches to adaptive thresholding for phone photos, but flat light is best.
  • Hold the phone square to the page (the sheet’s corner marks help).
  • Plain or lightly-ruled paper reads best.

4Getting the report to the visitor

Pick by how much setup you want. All keep the visitor’s data on the device unless you choose to host.

Save as PDF Easiest

Tap Save as PDF in the report toolbar, then AirDrop / email / WhatsApp the file. Works fully offline.

QR to a hosted link Recommended

Host the Studio (or a pre-rendered report) on any static host; print a QR on the sheet so visitors scan and view on their own phone.

Email capture Lead-gen

Use the optional email field on the intake screen, then send the PDF + a follow-up from go@vahinitech.com.

Make it a single file to share. The report can be exported as one self-contained HTML or a PDF, so a hosted link or an emailed file just works, no missing fonts or images.

5From demo to production

What to swap when the real hardware and models are ready.

  • Real recognition, set window.VAHINI_OCR_ENDPOINT to your PaddleOCR service. The app POSTs the image and uses the returned {lines:[{text,box,score}]} as the detected text automatically; offline it falls back to the local detector.
  • Real pen, the IMU screen streams a physically-plausible simulation today (no dataset yet). When the pen feeds live samples, swap the source in imu.js; the Kalman filtering, the §4C dynamics formulas and the report stay identical.
  • Calibration, the deterministic factors are measured now; the two estimated factors (legibility, character distinction) tighten once coach-rated samples are collected.
